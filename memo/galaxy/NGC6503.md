# NGC 6503 に関するメモ（中心 Newton 過大とデータ点数の問題）

## 1. 状態の概略

- SPARC rotmod から作った `build/NGC6503_sparc.csv` では、
  - 内側 R≲3 kpc にデータ点が **3 点**しかなく、
  - その範囲で Newton 合成曲線 \(V_{\rm Newt}\) が Vobs を最大 ~30 km/s 程度上回る。
- 現行の v2 カーネル（FDB2）では
  - HSB 銀河に対して中心部の星成分（Disk+Bulge）のスケーリング係数 \(s\) を推定し、
  - R≤min(3 kpc, 2Rd) のみを使って Newton≒Vobs になるように補正するロジックを追加した。
- しかし NGC 6503 では
  - 内側有効点数が 3、かつ星成分支配が弱いため、
  - 安全側の条件（有効点≥3、星成分優位）にギリギリ乗らず、
  - 2025-12-01 時点の実装では補正がスキップされている。

## 2. なぜ Newton が過大になっているか

- SPARC 質量モデルは 3.6µm 光度に一律の M/L を掛けるため、
  - HSB かつ小型ディスクの NGC 6503 では中心の星面密度が高く見積もられる傾向がある。
- rotmod の Vdisk（+Vbulge=0）だけで内側の \(V_{\rm circ}\) を作ると、
  - 中心 2〜3 kpc で Vobs よりも大きな円速度が出る。
- Vobs 側は
  - Hα/HI LOS 速度から「円運動成分」を抽出したものだが、
  - 中心部は非円運動やビームスミアの影響を強く受けており、
  - 実効の \(V_\phi\) が真の円速度よりも低く見える可能性がある。

## 3. 補正ロジックが効いていない理由

- v2 の `rescale_stellar_inner` は
  - HSB 銀河で
  - R≤R_cut_inner（min(3 kpc,2Rd)) に **3 点以上**の有効点があり、
  - かつ星成分 \(V_{\rm disk}^2+V_{\rm bul}^2\) が内側で有意に効いている場合にだけ
    \[
      V_{\rm Newt}^2 \to s\,(V_{\rm disk}^2+V_{\rm bul}^2) + V_{\rm gas}^2
    \]
    の形でスケーリングを行う。
- NGC 6503 では
  - 星成分支配が弱く
  - 有効点数ぎりぎりのため
  - 2025-12-01 時点の閾値設定では「補正を掛けない」側に分類されている。

## 4. 今後の扱いの方針

1. **短期**  
   - v2 フィットでは NGC 6503 の内側過大は「質量モデル＋内側運動学の系統」として扱い、FDB の評価は R>3R_d の外縁に絞る。
   - all.png などの総覧では、「中心 Newton 過大」がある HSB 銀河（NGC 1090, 2903, 3198, 3521, 5005, 5055, 6503, 7331, 7793）としてフラグを立てておく。

2. **中期**  
   - NGC 6503 のように内側データ点が少なくても Newton−Vobs の差が非常に大きいケースに限り、
     1 パラメータ \(s\) をより積極的に推定する「例外ルール」を導入する余地がある。
   - ただし、その場合は論文側で
     - 「SPARC rotmod の固定 M/L が中心で盛られている可能性」
     - 「内側の観測誤差（非円運動・ビームスミア）が eVobs に反映されていないこと」
     を明示し、補正の物理的妥当性を説明する必要がある。

3. **長期**  
   - NGC 6503 に関する最新の 2D 速度場（IFU/HI マップ）を確認し、
     - 内側の回転 vs 非円運動・分散の寄与
     - 星 M/L のより精密な推定
     を参照したうえで、新しいバリオン質量モデルを試すことも検討する。
