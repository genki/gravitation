# タスク終了時の通知ポリシー

目的: 作業の終了を関係者に即時共有し、結果と次アクションを
明確化する。

送信方法:
- スクリプト: `scripts/notice.sh`
- 既定: 非同期送信（`-a`未指定）。

タイトル（必須）:
- タスクの要約を簡潔に記述（全角30文字程度を目安）。

本文（推奨構成）:
- 結果: 1〜3行で要約
- 変更点/成果物: 主要ファイルや成果
- 次の一手: フォローアップや保留事項

Slack:
- `SLACK_WEBHOOK_URL`が設定されていれば同時投稿。

テンプレート:
```
タイトル: <タスク要約>
本文:
- 結果: <要点>
- 成果物: <主要ファイル>
- 次の一手: <予定/依頼>
```

備考:
- 同期応答が必要な場合のみアクションボタン`-a`を利用。
- 同期はheadless環境でブロックしやすいため通常は非推奨。
